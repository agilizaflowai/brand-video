# Captions Template — MAGNO

> Estruturas de legenda por pilar, com exemplos completos e regras de voz.
> Voz conforme `voice-tone.md`. Pilares conforme `content-pillars.md`.
> Ideias conforme `post-ideas.md`.

---

## Regras universais de caption (todos os pilares)

| Regra | O que fazer |
|---|---|
| **Abertura (hook)** | Frase afirmativa em até 10 palavras. Sem pergunta retórica tipo "Você sabia que...?" |
| **Primeira linha** | Deve funcionar sozinha — o LinkedIn/Instagram corta aqui se o usuário não expandir |
| **Linhas em branco** | Sim. Um parágrafo por ideia. Respiros são parte do peso editorial |
| **Hashtags** | **Zero** |
| **Emojis** | **Zero** |
| **Exclamação** | **Zero** |
| **Menção (@)** | Apenas em citação nominal real (ex: @veiculo-de-imprensa citou MAGNO) |
| **CTA** | Quase sempre ausente. Quando presente: imperativo seco. *"Ler carta completa →"*, *"Agendar conversa."* |
| **Assinatura** | Pode terminar com `— MAGNO` ou apenas com o bordão. Não obrigatório |
| **Comprimento** | LinkedIn: 800–1800 caracteres é o sweet-spot. Instagram: até 2200 |
| **Números** | Sempre com unidade explícita — *R$ 80M* (não "80 milhões"). Monoespaçada se tabular |
| **Tagline** | Usar Camada 1 (*Arquitetura antes de alocação.*) como fecho ocasional, não em toda legenda |

## Estrutura por pilar

### Pilar 1 — Princípios

**Estrutura canônica em 4 blocos:**

```
[BLOCO 1 — DECLARAÇÃO]
Uma frase curta, afirmativa, em negrito mental. É o bordão.

[BLOCO 2 — JUSTIFICATIVA]
1 parágrafo (2–4 linhas) explicando por que esse princípio existe.
Nem moral, nem didática — enunciação da lógica.

[BLOCO 3 — CONSEQUÊNCIA PRÁTICA (opcional)]
O que o princípio implica em operação real.

[BLOCO 4 — FECHO]
Bordão curto. Pode ser repetição do bloco 1 ou variação.
```

**Extensão:** 60–150 palavras.

---

### Pilar 2 — Análise

**Estrutura canônica em 5 blocos:**

```
[BLOCO 1 — LEAD COM DADO]
Primeira linha já traz o número relevante. É o hook.

[BLOCO 2 — CONTEXTO]
1–2 parágrafos explicando o mercado / prática / mecanismo.

[BLOCO 3 — DADOS ADICIONAIS]
2–3 números ou fatos. Citar fonte quando houver (Oliver Wyman,
BCG, CVM, Anbima, auditorias disponíveis).

[BLOCO 4 — IMPLICAÇÃO]
O que isso quer dizer em uma arquitetura patrimonial real.

[BLOCO 5 — FECHO]
Nem conclusão nem CTA — uma frase que enquadra sem moralizar.
```

**Extensão:** 180–450 palavras.

---

### Pilar 3 — Travessia

**Estrutura canônica em 4 blocos:**

```
[BLOCO 1 — CENÁRIO]
"Um founder vendeu 60 % da empresa em 2022..."
Cenário composto (nunca identificável). Em 1–3 linhas.

[BLOCO 2 — DILEMA]
Qual foi a decisão enfrentada, quais os trade-offs.

[BLOCO 3 — DESENLACE]
A decisão tomada + o princípio estrutural por trás.

[BLOCO 4 — GENERALIZAÇÃO DISCRETA]
O que esse tipo de situação costuma ensinar sobre arquitetura.
SEM moralizar. Sem "a lição é...".
```

**Extensão:** 200–400 palavras.

---

### Pilar 4 — Institucional

**Estrutura canônica em 3 blocos:**

```
[BLOCO 1 — FATO]
O que aconteceu / foi publicado. Direto, sem adjetivo.

[BLOCO 2 — CONTEXTO]
Por que isso existe, qual a lógica editorial / institucional.

[BLOCO 3 — ACESSO OU IMPLICAÇÃO]
Link / PDF / próximos passos. Se não houver, omitir bloco.
```

**Extensão:** 60–180 palavras.

### EX. 3 · Pilar 2 · Análise · "Aritmética silenciosa do rebate"

```
Em um portfólio de R$ 80M, 1,3 % a.a. são R$ 1M por ano que não aparecem no extrato.

O rebate embutido em produtos estruturados é, em média, entre 1 % e 3 % do notional,
distribuído silenciosamente pela cadeia de comissão — distribuidor, plataforma, gerente.

Três pontos que ajudam a entender a aritmética:

— O rebate não aparece na taxa de administração nominal. Ele está precificado dentro do
produto antes da cotação de mercado que o cliente vê.

— Em produtos com prazo, o custo é anualizado. Um COE de 5 anos com 2 % de rebate total
é 0,4 % a.a. invisível.

— Em dez anos, em um portfólio de R$ 80M com exposição média à distribuição, a perda
acumulada está na ordem de R$ 11,9M — sem impacto composto.

A matemática do custo oculto é silenciosa. Uma arquitetura independente vê esse custo.
Um canal de distribuição não tem mandato para ver.

Fonte dos ranges de rebate: levantamentos da Anbima e de auditorias internacionais
comparáveis em mercados maduros (UK, US pré-DOL rule).
```

### EX. 4 · Pilar 2 · Análise · "Holding: quando faz sentido"

```
Holding patrimonial é um bom instrumento. Para a pessoa errada, é custo sem proteção.

Antes de montar uma holding, três perguntas objetivas:

01 · O patrimônio líquido total supera R$ 20M? Abaixo disso, o custo fixo anual
(contabilidade, auditoria, declarações) tende a superar o ganho estrutural.

02 · Há patrimônio produtivo relevante (quotas de empresa operacional, imóveis
comerciais, participação em fundos de investimento de longo prazo)? Holding faz mais
sentido para ativos que geram renda do que para patrimônio parado.

03 · Há arquitetura de sucessão a ser desenhada entre múltiplos herdeiros? Se sim, a
holding é ferramenta central.

Se a resposta for "não" para as três, holding é custo sem contrapartida estrutural.

A decisão de montar holding não é estética. É aritmética sobre ativos, fluxos e
sucessão. E precede qualquer conversa sobre alocação.

— MAGNO
```

### EX. 5 · Pilar 3 · Travessia · "O ano depois do M&A"

```
Doze meses depois do M&A, o founder costuma descobrir três coisas que ninguém
mencionou na due diligence.

A primeira: o fluxo de caixa pessoal se inverte. Por quinze ou vinte anos, o salário foi
apenas um dos canais — e com frequência não o principal. A renda variável vinha de
distribuição de lucros, reembolsos, benefícios societários. Depois do evento, o fluxo
precisa ser desenhado do zero, e com uma horizonte diferente (25 anos vs 2–3).

A segunda: a sucessão, que até então era tema "para depois", vira urgente. O patrimônio
antes estava diluído entre equity ilíquido e compromissos operacionais. Agora está
concentrado em liquidez — e liquidez é o que gera disputa em qualquer família.

A terceira: o círculo social se altera silenciosamente. Fornecedores, ex-sócios,
amigos próximos do tempo de construção — todos começam a se relacionar de uma forma
levemente diferente. Isso é humano. É esperado. E faz parte do que se estrutura.

MAGNO desenha arquitetura para esse momento específico. O que muda nos primeiros doze
meses é exatamente o que não pode ser tratado como tema isolado — é o sistema todo,
ao mesmo tempo.

A travessia é estrutural. A resposta também tem que ser.
```

### EX. 6 · Pilar 4 · Institucional · "Carta trimestral"

```
Publicamos hoje a primeira carta trimestral institucional da firma.

A carta é pública, acesso livre, sem cadastro. Leitura de cerca de 20 minutos. Cobre três
frentes:

01 · Fluxos institucionais de rebate e seu impacto composto em portfólios brasileiros
02 · Arquitetura holding 2026: o que mudou na tributação de dividendos
03 · Drawdown histórico em composições conservadoras brasileiras (séries 2005–2025)

A decisão de publicar cartas decorre da mesma lógica da firma: arquitetura é tese,
e tese precisa estar disponível, auditável e argumentável — não restrita a cliente.

Ler a carta → magno.capital/cartas/q1-2026

— MAGNO
```

## Como exportar os templates como PNG (Playwright)

Cada template tem `id` único (t01 → t10) e dimensões exatas. Para gerar PNGs:

```bash
# Requisitos: Node.js + Playwright
npm install -g playwright
npx playwright install chromium
```

```js
// scripts/export-templates.js
const { chromium } = require('playwright');
const fs = require('fs');
const path = require('path');

const TEMPLATES = [
  { id: 't01', w: 1080, h: 1080 },
  { id: 't02', w: 1080, h: 1350 },
  { id: 't03', w: 1080, h: 1350 },
  { id: 't04', w: 1080, h: 1350 },
  { id: 't05', w: 1080, h: 1080 },
  { id: 't06', w: 1080, h: 1920 },
  { id: 't07', w: 1080, h: 1920 },
  { id: 't08', w: 1080, h: 1080 },
  { id: 't09', w: 1200, h: 627  },
  { id: 't10', w: 1080, h: 1920 },
];

(async () => {
  const browser = await chromium.launch();
  const ctx = await browser.newContext({ deviceScaleFactor: 2 });
  const page = await ctx.newPage();
  const file = 'file://' + path.resolve(__dirname, '../output/06-social-kit/templates.html');
  await page.goto(file);
  await page.waitForLoadState('networkidle');

  for (const t of TEMPLATES) {
    const el = await page.locator(`#${t.id}`);
    await el.screenshot({ path: `output/06-social-kit/exports/${t.id}.png` });
    console.log(`✓ ${t.id}.png — ${t.w}×${t.h}`);
  }

  await browser.close();
})();
```

Rodar:
```bash
mkdir -p output/06-social-kit/exports
node scripts/export-templates.js
```

Cada PNG sai em resolução 2× (retina) — pronto para publicação em LinkedIn, Instagram,
Twitter. Se precisar 1× (tamanho menor), remover `deviceScaleFactor: 2`.

## Checklist antes de publicar qualquer post

- [ ] Caption passa no teste das 7 princípios do `voice-tone.md`?
- [ ] Zero palavra proibida (lista em `voice-tone.md` §7)?
- [ ] Primeira linha funciona sozinha (corte do feed)?
- [ ] Zero emoji, zero exclamação, zero hashtag?
- [ ] Se tem número: unidade explícita e tratamento Plex Mono em visual?
- [ ] Se tem cenário (Pilar 3): cliente real não identificável?
- [ ] Visual está em um dos templates aprovados (t01–t10)?
- [ ] Link canônico é `magno.capital/...`?
- [ ] Tempo de publicação: 2ª, 4ª ou 6ª, entre 8h30 e 10h?
- [ ] Aprovação final do partner-fundador registrada?
